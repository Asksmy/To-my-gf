# PROJECT: HOME
## Interactive Emotional Experience
### Version 1.0

---

# 0. VISION STATEMENT

PROJECT: HOME is not a website.

It is an interactive emotional experience designed to transform a shared love story into a living universe that can be explored.

It is not meant to impress technically.

It is meant to be remembered emotionally.

The experience should make two people feel closer, regardless of physical distance.

Every interaction is a memory.
Every memory becomes part of the sky.
Every sky reflects a relationship that is still being built.

---

# 1. CORE PRINCIPLE

Emotion > Beauty  
Meaning > Spectacle  
Silence > Noise  
Curiosity > Instruction  
Presence > Performance  
Honesty > Perfection  

If any technical decision conflicts with emotional clarity, emotional clarity always wins.

---

# 2. EXPERIENCE OVERVIEW

Total duration: 6–8 minutes

Structure:

1. The Universe Awakens  
2. The Sky Begins To Grow  
3. Memories Become Constellations  
4. The Sky They Built Together  
5. The Quiet Ending  

The pacing is slow, intentional, and reflective.

Nothing is rushed.

---

# 3. EMOTIONAL GOAL

The user should feel:

- Safe
- Calm
- Loved
- Present
- Hopeful

Not overwhelmed.
Not manipulated.
Not forced.

If tears happen, they must come naturally from understanding, not pressure.

---

# 4. INTERACTION PHILOSOPHY

There are no:

- Levels
- Objectives
- Scores
- Progress bars
- Tasks

Only discovery.

Every interaction should feel like curiosity, not completion.

The user explores because they want to understand, not because they are told to.

---

# 5. THE UNIVERSE

The universe is not a narrator.

It is an observer.

It speaks rarely.

When it speaks, it uses simple, timeless sentences.

It never explains emotions.

It only reflects them.

Example tone:

- “Some skies are built slowly.”
- “Not everything that shines is distant.”
- “Some stories begin before they are noticed.”

---

# 6. STARS & CONSTELLATIONS

Stars represent:

- memories
- conversations
- promises
- emotions
- shared moments

Each star is unique.

Stars do not behave like UI elements.

They behave like living light.

Constellations are formed through interaction, not static design.

Light connects them gradually.

---

# 7. TWO MAIN STARS

Two primary stars exist in the sky.

They represent the two people in the relationship.

They evolve subtly across the experience.

They slowly move closer through:

- memories unlocked
- emotional progression
- user interaction

This movement is almost imperceptible until the end.

---

# 8. MEMORY SYSTEM (DATA-DRIVEN DESIGN)

All memories must be defined as structured data.

Example:

```json
{
  "title": "3 AM Conversations",
  "emotion": "nostalgia",
  "visual": "clock_stars",
  "colors": ["#FFD79A", "#FFF5E8"],
  "narration": "Time disappeared when talking to you.",
  "effects": ["slow_camera_zoom", "star_connection", "soft_music_layer"]
}

No memory should be hardcoded in logic.

Everything must be configurable.

9. VISUAL DESIGN
Palette
Deep space black
Warm white
Soft gold
Dark navy blue
Sunset orange (rare)
Dusty pink (emotional moments)
Wine red (intense memories)
Principles
No neon colors
No harsh contrast
No overly saturated visuals
Everything should feel soft and emotional
Atmosphere
Floating particles (subtle, not distracting)
Gentle nebulae (low opacity)
Soft light bloom
Depth layers (foreground / mid / background)
10. MOTION DESIGN

Nothing moves at constant speed.

Everything:

breathes
accelerates gently
slows naturally

Rules:

No sudden transitions
No linear motion
No mechanical animation

Animation must feel alive, not programmed.

11. AUDIO DESIGN

Audio is not background decoration.

It is emotional structure.

Piano-based themes
Ambient space sound
Silence as intentional tool

Music must evolve gradually without noticeable transitions.

Silence is part of composition.

12. NARRATION DESIGN

Narration appears rarely.

Text must:

fade in gently
feel like discovery
never interrupt emotion

No long paragraphs.

No explanations.

Only essential sentences.

13. CAMERA SYSTEM

The camera behaves like curiosity.

It:

floats slightly
moves slowly
never feels robotic
reacts softly to interaction

Transitions are smooth and continuous.

14. SCENE STRUCTURE

Each scene must be independent:

Initialization
Interaction
Emotional beat
Transition out

Scenes do not depend tightly on each other.

They are modular experiences.

15. TECHNICAL ARCHITECTURE

Stack:

HTML5
CSS3
JavaScript (ES6 Modules)
GSAP (animations)

Optional:

Audio API for sound control

Avoid unnecessary complexity (e.g., Three.js unless required).

Core Modules
ExperienceManager
SceneManager
AnimationController
CameraController
AudioManager
StateManager
InteractionManager

Each module has a single responsibility.

Data-Driven System

All content must be stored in JSON:

memories.json
narration.json
theme.json

No hardcoded emotional content inside logic.

16. PERFORMANCE RULES

Target:

60 FPS stable

If any effect reduces performance:

simplify it
remove it
replace it

Emotional continuity is more important than visual complexity.

17. ACCESSIBILITY
Reduced motion support
Readable typography
High contrast where needed
No mandatory interaction timing
Mobile-first responsiveness
18. EMOTIONAL RULES (HARD CONSTRAINTS)
Never force emotional reaction
Never try to “make user cry”
Never over-explain feelings
Never use dramatic exaggeration
Always prioritize subtlety

Hope must always exist.

19. DEVELOPMENT PHASES
Core foundation
Universe (sky, stars, camera)
Interaction system
Memory system
Narrative layer
Audio integration
Final letter
Polish & optimization

Never build everything at once.

20. FINAL PRINCIPLE

If all visuals were removed, but the emotional feeling remained:

The project succeeds.

If visuals are perfect but emotion is lost:

The project fails.
